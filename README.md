# 5312_FinalProject

This is a reanalysis of the ball python genome assembly published in 2025. The original study is under bioproject PRJNA1217506 on NCBI. The goal of the original study was to construct a novel genome assembly for the species Python regius (Ball python). Pythons are a good model to study extreme metabolic and organ structural plasticity due the unique nature of significant organ remodeling, metabolic alterations, and associated changes in gene expression involved in their feeding cycle. 

The aim of this reanalysis will be to broadly reproduce the genome assembly using publicly available sequencing data and standard bioinformatics tools provided by the University of Connecticut HPC. While the original study reported the tools used, it did not include fully reproducible code or pipelines. As a result, this project reconstructs the workflow using commonly accepted methods in genome assembly and polishing, with an emphasis on transparency and reproducibility. Although some differences in tool choice and parameterization exist, the overall structure of the pipeline closely follows the original study’s approach.

The workflow begins with the acquisition of raw sequencing data from NCBI, including both Illumina paired-end reads and Oxford Nanopore long reads. Illumina reads are processed using BBDuk to remove adapter sequences and low-quality bases. The genome is then assembled using long-read data with Flye, which provides the structural backbone of the assembly. This initial assembly is subsequently polished in multiple stages: first using Medaka for long-read polishing, followed by Pilon for short-read polishing to improve base-level accuracy.

Following assembly and polishing, both Illumina and Nanopore reads are aligned back to the genome using BWA-MEM and Minimap2, and processed with Samtools to generate sorted and indexed BAM files. These alignments are used in a contamination assessment step using BlobToolKit, where sequence origin is evaluated based on coverage and taxonomic classification. Contigs that do not meet quality or length thresholds are removed to produce a filtered assembly.

The final assembly is evaluated using standard quality assessment tools. Genome completeness is assessed using BUSCO, which searches for conserved single-copy orthologs, while general assembly statistics such as contig length and N50 are calculated using QUAST. Together, these metrics provide a comprehensive assessment of the reconstructed genome.

All tools used in the pertaining scripts are provided by UConn HPC, with the exception of Blobtoolkit. This software was run through a custom Conda environment named blobtk39 in the script. This environment was created with Python 3.9 and BlobToolKit was installed using pip, which avoided dependency conflicts. Attempts to install BlobToolKit into an existing environment caused dependency conflicts and missing package errors. Creating a fresh environment and installing BlobToolKit with pip provided a smore reliable installation for this workflow.
